## ¿Qué es ESLint?

ESLint es un linter de código abierto para JavaScript. Un linter es una herramienta de análisis de código que ayuda a detectar errores y problemas en el código, su objetivo principal es mejorar la calidad del código al asegurarse de que sigue ciertos estándares.

### Utilidades de un Linter

- Ayuda a detectar errores: un linter puede detectar errores comunes de programación, como variables no definidas, errores de sintaxis, problemas de formato de código, entre otros.

- Mejora la calidad del código: al detectar y corregir errores, el código se vuelve más fácil de entender y mantener.

- Ayuda a mantener un estilo consistente, como el uso de comillas simples o dobles, espacios o tabs, entre otros.

## ¿Qué es Standard?

Standard es un paquete de node que trae consigo un conjunto de reglas y buenas prácticas para seguir en JavaScript. Las reglas son configurables, pero por defecto Standard viene con un conjunto de reglas que se ajustan a la mayoría de los proyectos.

## Instalación

Hay varias formas de utilizar ESlint y Standard, acá mencionaré dos, usando ESLint y Standard y usando solamente StandardJS, en lo personal prefiero usar ESLint y Standard, ya que me permite tener más control sobre el linter y ajustarme a proyectos que no usen StandardJS.

### Usando ESLint y Standard

1. Instalar la extensión de [ESLint](https://marketplace.visualstudio.com/items?itemName=dbaeumer.vscode-eslint) en VSCode.

2. Abrir command palette `Ctrl + Shift + P` y buscar `Open User Settings (JSON)`.

3. añadir las siguientes líneas al archivo `settings.json`:

    ```json 
    {
      "editor.codeActionsOnSave": {
        "source.fixAll.eslint": true
      }
    }
    ```

4. Dentro del proyecto instalar standard `npm install standard -D` o `yarn add standard -D`.

5. Dentro de `package.json`añadir las siguientes líneas:

    ```json
    {
      "eslintConfig": {
        "extends": "standard"
      }
    }
    ```

### Usando solo StandardJS

1. Instalar la extensión de [StandardJS](https://marketplace.visualstudio.com/items?itemName=standard.vscode-standard) en VSCode.

2. Abrir command palette `Ctrl + Shift + P` y buscar `Open User Settings (JSON)`.

3. añadir las siguientes líneas al archivo `settings.json`:

    ```json 
    {
      "standard.autoFixOnSave": true,
    }
    ```

4. Dentro del proyecto instalar standard `npm install standard -D` o `yarn add standard -D`.

## Configuración adicional

1. Importante desactivar la opción `Format On Save`, ya que StandardJS y ESLint tienen sus propias opciones y activarlo puede causar conflictos.

2. Si usas prettier ya no es necesario, porque StandardJS y ESLint corregirán los errores al guardar.

Estos metodos también funcionan para otras variantes de `standard` como `semistandard` o `ts-standard`.
